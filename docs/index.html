AppImage

One file to rule them all

The Deb file was designed to use Ubuntu libraries, which is very unconvinient for non-Ubuntu distribution as it forces to tweaks in a dirty way with the libraries and messing them up.

Alex^#1629 previously built an AppImage as Blade was not providing one. They started to build one, so the AppImage is no longer available to avoid duplicity.

The previous versions are still available on Alex's repository but will be not updated at all. We strongly recommand you to now download the application from the Official Website only.
The download button will directly point to the official website.

Even if the AppImage is now available on the official website, this website is still updated as the tutorial are still up to date.

Linux Mint 19 or higher Linux Mint 18

To use it, just download the AppImage, put it where you want, make it executable and then launch it.

Useful information

For Arch Linux users, you need to install the libsndio-61-compat.

For Fedora users, you may check that you have installed the library librtmp.

For Debian or Ubuntu users with CPU Gen 8+, you may set an environment variable LIBVA_DRIVER_NAME=iHD or start the AppImage like this LIBVA_DRIVER_NAME=iHD ./Shadow.AppImage --no-sandbox.

For every user who wants the "remember me" feature, you need to install gnome-keyring

Setup your GPU for the VA-API

As Shadow for Linux uses the VA-API, it is important to well setup your GPU for it.
So it is important to understand how works vainfo and how to debug it.

First things first, vainfo needs to be installed on your OS. It is a program that describes how your GPU is detected by the VA-API.

  • Debian based distribution: sudo apt install libva-glx2 vainfo
  • Arch Linux: sudo pacman -S libva-utils
  • Solus: sudo eopkg it libva-utils

The vainfo when executed in a terminal will return the abilities of you GPU. There are 3 things to check:

  • If it returns nothing or returns -1 somewhere, it means that the GPU is not well detected. Your VA driver needs to be updated.
  • In the list of profiles, it mentions "H264" at least once. If supported, you can use the Shadow on Linux.
  • In the list of profiles, it mentions "H265" or "HEVC" at least once. If not, you will not be able to use H265 but as long as you have H264, you still can use Shadow on Linux.

Install the drivers

Even if Xorg recognizes your GPU, VA-API sometimes needs specific drivers to use it. The best way to install them is to follow the official documentation of your distribution about it:

  • Debian based
  • Arch Linux
You may also search in the repositories the name of your GPU with libva. For instance, on Solus with an Intel, sudo eopkg it libva-intel-driver.

Always check your vainfo. You do not need to reboot after the installation of the drivers.

For most cases

If your active GPU is an Intel chipset and you have no dedicated GPU, you will need the i965 Intel driver.

  • Ubuntu/Debian: sudo apt install i965-va-driver
  • Ubuntu/Debian with CPU Gen 8+ : sudo apt install intel-media-va-driver-non-free
  • Arch Linx: sudo pacman -S libva-intel-driver
  • Solus: sudo eopkg it libva-intel-driver

For NVIDIA users, you have 2 options:

  • Use the Nouveau driver and install the old NVIDIA firmware that provides support for the VA API.
  • Use the Arekinath patch to provides VA API compatibility for recent NVIDIA cards.

In both cases, we strongly recommend you to follow this link: https://gitlab.com/aar642/shadowos-boot#support
